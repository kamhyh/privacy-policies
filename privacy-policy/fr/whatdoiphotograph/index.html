Politique de confidentialité

Where Do I Take Photograph (WhatDoIPhotograph) — application Android

1. Collecte des données

Where Do I Take Photograph collecte les données suivantes lorsque vous l'utilisez :

  • Données de localisation (latitude, longitude, horodatages) : collectées pendant qu'une session d'enregistrement est active, afin de tracer votre itinéraire photographique et de marquer les spots où vous vous arrêtez pour photographier.
  • Photographies que vous choisissez d'attacher à un spot : stockées sous forme de fichiers JPEG dans le stockage privé de l'application.
  • Métadonnées de session : titre de la session, durée, distance, notes, évaluations, et les associations entre spots et collections que vous créez.
  • Télémétrie de crashs et d'événements : rapports de crashs anonymisés et événements d'usage basiques (vues d'écran, interactions de fonctionnalités) envoyés à Firebase Crashlytics et Firebase Analytics afin de diagnostiquer les problèmes et de comprendre quelles fonctionnalités sont utilisées.

2. Stockage local

Tout le contenu utilisateur principal — sessions, points d'itinéraire, spots, photographies et collections — est stocké localement sur votre appareil dans une base de données SQLite (Room) à /data/data/com.photosession.app/databases/ et dans le stockage de fichiers privé de l'application à /data/data/com.photosession.app/files/spot_photos/. Aucun de ces contenus n'est transmis à un serveur distant à moins que vous ne le partagiez ou ne l'exportiez explicitement.

L'application n'a pas de serveur distant pour le contenu généré par l'utilisateur.

3. Autorisations

L'application demande les autorisations Android suivantes :

  • ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION : requise pour enregistrer les coordonnées GPS lorsqu'une session est active.
  • ACCESS_BACKGROUND_LOCATION : Nécessaire pour la continuité de l'enregistrement — l'application enregistre des sessions photographiques basées sur la localisation qui peuvent continuer lorsque l'utilisateur quitte l'application ou éteint l'écran. Sans la localisation en arrière-plan, la trace GPS de la session serait perdue.
  • FOREGROUND_SERVICE / FOREGROUND_SERVICE_LOCATION / FOREGROUND_SERVICE_DATA_SYNC : requises pour maintenir le service d'enregistrement en arrière-plan et pour exécuter le service d'export/import comme opération de premier plan avec une notification visible.
  • POST_NOTIFICATIONS (Android 13+) : requise pour afficher la notification d'enregistrement en cours et la notification de progression de sauvegarde.
  • INTERNET : requise pour récupérer l'image de carte statique utilisée lors du rendu de l'image de partage (voir §4).

4. Services tiers

L'application utilise les services tiers suivants :

  • Google Maps SDK et Google Static Maps API (Google LLC) : affiche la carte à l'intérieur de l'application et rend la tuile de carte statique incluse dans la génération de l'image de partage. La latitude/longitude de l'itinéraire est transmise aux serveurs de Google lors du rendu de l'image de partage. La politique de confidentialité de Google s'applique : https://policies.google.com/privacy.
  • Firebase Crashlytics (Google LLC) : reçoit des rapports de crashs anonymisés (traces d'appels, modèle d'appareil, version du système, version de l'application). Aucune information personnellement identifiable n'est intentionnellement envoyée. Traitement des données de Crashlytics : https://firebase.google.com/support/privacy.
  • Firebase Analytics (Google LLC) : reçoit des événements d'usage anonymisés (vues d'écran, interactions de fonctionnalités). Aucun identifiant publicitaire n'est collecté ; les identifiants publicitaires ne sont explicitement PAS utilisés par cette application.

La collecte par Crashlytics et Analytics est désactivée dans les builds de debug. Les builds de production collectent la télémétrie par défaut ; le contrôle de désinscription en v1 est la désinstallation de l'application.

5. Partage des données

L'application partage des données uniquement avec les tiers nommés au §4 (Google Maps API, Firebase Crashlytics, Firebase Analytics) et uniquement aux fins décrites ci-dessus. L'application ne vend ni ne loue les données des utilisateurs à aucune partie. L'application ne diffuse PAS de publicités et ne collecte PAS d'identifiant publicitaire.

6. Vos droits

Vous avez le contrôle total de vos données :

  • Exporter vos données via Paramètres → DONNÉES → Exporter. L'application produit une seule archive .zip contenant chaque session, point d'itinéraire, spot, photographie, collection et association, exportée via le Storage Access Framework Android vers un emplacement de votre choix.
  • Importer vos données sur le même appareil ou un autre via Paramètres → DONNÉES → Importer. L'archive est ajoutée (additif) ; les enregistrements existants sont préservés.
  • Supprimer vos données en désinstallant l'application. Toutes les données stockées sur l'appareil sont supprimées par le système d'exploitation lors de la désinstallation.
  • L'application ne fait pas tourner de serveur distant, il n'y a donc aucun compte distant à supprimer ni aucune donnée distante dont demander une copie.

7. Confidentialité des enfants

L'application est classée « Tout public » / PEGI 3 et ne collecte pas sciemment de données auprès de personnes de moins de 13 ans. Elle ne nécessite pas la création de compte, ne collecte pas d'informations personnelles (nom, e-mail, numéro de téléphone) et ne contient pas de publicité.

8. Modifications

Cette politique de confidentialité peut être mise à jour lorsque l'application évolue. La date d'effet ci-dessous reflète la dernière révision. Les changements significatifs seront mentionnés dans les notes de version de l'application (« Quoi de neuf » sur le Play Store).
